# Branch Strategy

## Branches
- main — stable, production-ready code
- dev — active development, all features merge here first
- feature/ticket-name — one branch per feature, created from dev

## Workflow
1. Create a feature branch from dev
2. Work on your feature and commit regularly
3. Push and open a Pull Request into dev
4. Merge into dev once complete
5. dev gets merged into main at each milestone

## Branch Naming
feature/short-description
Example: feature/wallet-connection
