## Project Overview

A fullstack dApp where users connect MetaMask, create proposals, and vote on-chain. Proposals and votes are stored in a Solidity smart contract on Sepolia testnet; off-chain metadata lives in a PostgreSQL database (Neon cloud-hosted).

## Repository Layout

```
/                  # Hardhat project root — contracts, scripts, tests
├── contracts/     # Solidity source files (Voting.sol, Lock.sol)
├── artifacts/     # Auto-generated by Hardhat compile (do not edit)
├── cache/         # Hardhat compile cache (do not edit)
├── frontend/      # Next.js 14 app (App Router)
└── backend/       # Express.js API server
```

> Note: Despite the README mentioning a `blockchain/` directory, Hardhat lives at the repo root. Run all Hardhat commands from `/`.

## Commands

### Blockchain (run from repo root)
```bash
npx hardhat compile       # compile contracts → artifacts/
npx hardhat test          # run all contract tests
npx hardhat test --grep "description"  # run a single test by name
```

### Frontend
```bash
cd frontend
npm install
npm run dev               # starts on http://localhost:3000
```

### Backend
```bash
cd backend
npm install
npm run dev               # nodemon server, default port 5000
```

## Smart Contract

**File:** `contracts/Voting.sol`  
**Deployed:** Sepolia testnet at `0x0eE97B60A88421E106e4999EBCF3D0144b479A94` (block 11113560)

Key details:
- `createProposal(title, description, durationInMinutes)` — duration is in minutes, converted to deadline timestamp on-chain
- `vote(proposalId, voteYes)` — one vote per address per proposal; enforced via `hasVoted[proposalId][addr]` mapping
- `getProposal` / `getAllProposals` — gas-free view functions
- Proposals are 1-indexed (`proposalCount` starts at 0, increments before assignment)
- Events: `ProposalCreated(id, title, creator, deadline)` and `VoteCast(proposalId, voter, support)`

## Frontend Stack

Next.js 14 (App Router) + TypeScript + TailwindCSS + wagmi + viem + ethers.js

wagmi/viem are the primary wallet abstraction layer. ethers.js is also installed for direct contract interactions where needed.

**Frontend source layout** (`frontend/src/`):
```
app/                        # Next.js App Router pages
  page.tsx                  # Home (hero + example proposals)
  proposals/
    page.tsx                # Proposal list (requires wallet)
    create/page.tsx         # Create form (requires wallet)
    [id]/page.tsx           # Proposal detail + vote buttons
components/
  Navbar.tsx  Footer.tsx  Loading.tsx  Toast.tsx  WalletGuard.tsx
  ProposalCard.tsx          # Reusable card with yes/no vote bar
  Providers.tsx             # WagmiProvider + QueryClientProvider
hooks/useWallet.ts          # Wraps useAccount/useConnect/useDisconnect
lib/
  wagmi.ts                  # createConfig with sepolia + injected connector
  contract.ts               # ABI + contract address constant
services/
  blockchainService.ts      # getAllProposals, getProposal, createProposal, castVote
  apiService.ts             # Backend REST calls (metadata)
store/
  useProposalStore.ts       # Zustand store for proposals
  useUIStore.ts             # Toast/loading UI state
utils/proposal.ts           # mapProposalForUI, formatVoteError
```

`WalletGuard` wraps every page that needs a wallet — it renders "MetaMask not found" in headless environments (expected behavior during agent testing).

## Environment Variables

**Frontend** (`frontend/.env.local`):
```
NEXT_PUBLIC_CONTRACT_ADDRESS=   # deployed Voting contract address
NEXT_PUBLIC_BACKEND_URL=        # backend API base URL
```

**Backend** (`backend/.env`):
```
PORT=                   # default 5000
DATABASE_URL=           # PostgreSQL connection string (Neon)
ALCHEMY_URL=            # Alchemy Sepolia RPC endpoint
CONTRACT_ADDRESS=       # deployed Voting contract address
CONTRACT_DEPLOY_BLOCK=  # block number for log filtering
FRONTEND_URL=           # for CORS allowlist
```

**Blockchain** (root `.env`):
```
PRIVATE_KEY=            # deployer wallet private key
SEPOLIA_RPC_URL=
BASE_SEPOLIA_RPC_URL=
```

## Git Workflow

- `main` — stable, production-ready
- `dev` — active development, all features merge here first
- `feature/<short-description>` — one branch per feature, branched from `dev`, PR back to `dev`
- `dev` is merged into `main` at each milestone

Never commit `.env` files or private keys.
